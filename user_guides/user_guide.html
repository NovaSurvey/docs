NovaSurvey Console Guide

Welcome to the complete user guide for the NovaSurvey Interactive Console (`nova_console.exe`). Below you will find interactive walkthroughs of the console lifecycle, detailed parameter definitions for all 12 platform-native statistical engines, and 100% realistic runnable execution commands.

✨ Zero-Loading Instant Demo Mode

Want to test NovaSurvey immediately without generating or loading any datasets? Simply type demo <module> (e.g., demo outliers) directly into the interactive console! The console will automatically construct a schema-aligned 200-row demo dataset in-memory and execute the module instantly!

🚀 Interactive REPL Lifecycle

1
Generate Mock Testing Dataset
Run the comprehensive mock script in your terminal to create a perfectly schema-aligned testing file:
python create_comprehensive_sample.py
2
Launch Interactive Console
Launch the native console binary from your powershell prompt:
.\dist\nova_console.exe
3
Load Mock Dataset
Once inside the interactive `nova>` prompt, load the testing dataset into active memory:
load comprehensive_sample.parquet
4
Generate R-like Summaries & Columns
List columns or check exact statistical distributions across all variables:
cols
summary
5
Execute Statistical Runs
Copy any of the 100% runnable examples below to execute calculations instantly. Active state changes are kept in-memory and chained automatically!
6
Export Mutated Dataset
Save all your calculated survey weights, imputations, and seasonal adjustments back to disk:
export completed_results.parquet

Statistical Engine Documentation

Outlier Detection

editing

Performs robust outlier detection to flag extreme survey values for selective imputation using various statistical algorithms.

Parameters & Options

REPL Parameter CLI Flag Type Required Description
target --target Column Name Required The name of the numeric variable to check for extreme values.
method --method String Optional Detection method. Options: 'hb' (Huber's Method) [default], 'sigma-gap', 'iforest' (Isolation Forest), 'knn', 'lof'.
threshold --threshold Float Optional Deviation/cutoff threshold [default: 3.0].
prev_column --prev_column Column Name Optional Previous wave/period value column to match and compare against.
group_var --group_var Column Name Optional Stratum or grouping column for subgroup analysis.
weight_var --weight_var Column Name Optional Design weights column for weighted outlier detection.
critical_threshold --critical_threshold Float Optional Score threshold for marking an outlier as critical.

Statistical Manual

For full algebraic derivations, sample budgets, or validation formulas, view the local manual.

editing/editing_manual.md
📘 Open Markdown Manual

Runnable Examples

Interactive GUI Console (Python/R Syntax)
outliers(revenue, method="hb", threshold=2.5)
Batch Pipeline CLI Command
run outliers --target revenue --method hb --threshold 2.5
Zero-Loading Demo Command (Run Instantly!)
demo outliers

Nonresponse Weighting Adjustment

weighting

Adjusts initial survey design weights to compensate for nonresponse propensity using response homogeneity groups or propensity scores.

Parameters & Options

REPL Parameter CLI Flag Type Required Description
target_col --target_col Column Name Required Response indicator column (1=Response, 0=Nonresponse).
feature_cols --feature_cols String Required Comma-separated list of predictor features/covariates.
design_weight_col --design_weight_col Column Name Optional Column containing baseline survey design weights.
method --method String Optional Propensity adjustment method: 'rhg' (Response Homogeneity Groups) [default] or 'ipw'.
n_groups --n_groups Integer Optional Number of propensity strata/groups for RHG [default: 5].
weight_cap --weight_cap Float Optional Maximum weight adjustment factor to prevent extreme weight inflation [default: 3.0].
